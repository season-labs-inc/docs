## What is Era?

Era gives your AI agent memory of who the user is — their intent patterns, sentiment, history, and behavioral signals — without you having to build any of it. You call `turnContext()` once per turn and inject the returned block into your system prompt. That's it.

The block is computed by the Era service from your conversation history and looks like this:

```
[STATE v1]
user_id: u_abc123
lifecycle: engaged
motivation: validator
depth: deep
sentiment: positive
intent_resolved_rate: 0.72
top_intents: ticket purchase, price comparison
[/STATE]
```

Your agent reads this context and responds accordingly — personalizing tone, anticipating follow-ups, and avoiding repeating ground already covered.

## How it fits into your stack

```
User message
     │
     ▼
era.turnContext(messages)   ←── Era service (non-blocking in async mode)
     │
     ▼
Inject block into system prompt
     │
     ▼
LLM call  →  Response to user
```

Era runs **before** your LLM call. In `async` mode (the default) it returns instantly with the previous turn's precomputed block, so there is zero added latency on the critical path.
