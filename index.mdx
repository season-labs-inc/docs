## What is Era?

Era gives your AI agent a live read on the user. Once per turn, you call `turnContext()` and get back a short text block. You append it to your system prompt. That's the whole integration.

The block looks like this:

```
[STATE v1] urgency=immediate knowledge=uncertain_about_qualifying_format [/STATE]
[GOAL v1] active=seek:confirm_if_qualifying_already_happened resolved=unresolved [/GOAL]
[INTENT v1] intent=validating(0.95) specificity=specific journey=[exploring,learning,validating] [/INTENT]
[CAUSAL v1] trigger=fear_of_missing_session emotion=anxious need_level=safety [/CAUSAL]
[REC v1] action=validate do="Confirm immediately whether they missed qualifying" priority=0.85 [/REC]
```

With this in its prompt, your agent stops guessing. It knows the user is anxious, knows what they're really asking, and knows what to do — before it writes a word. [See a real conversation with and without it](/use-cases/fan-companion).

Era is not a memory system. Memory stores facts about a user — "has a dog named Rex." Era reads the conversation as it happens and tells your agent what the user wants *right now*. If you already use a memory layer, keep it; the two work side by side.

## Core concepts

| Concept | Description |
| --- | --- |
| **Block** | The text Era returns each turn. You append it to your system prompt. |
| **Signals** | What Era reads from the conversation: intent, goal, urgency, emotion, values. |
| **Recommendation** | The `[REC]` line — what Era suggests your agent do on this turn. |
| **Modes** | `async` (default) returns instantly with the previous turn's block. `sync` waits for a fresh one. |

## How does it work

```
User message
     │
     ▼
era.turnContext(messages)   ←── Era service (non-blocking in async mode)
     │
     ▼
Inject block into system prompt
     │
     ▼
LLM call  →  Response to user
```

1. **Your user sends a message.** Your app receives it as usual.
2. **You call `era.turnContext(messages)`.** In async mode this returns instantly with the block computed from the previous turn — nothing waits on Era.
3. **You append the block to your system prompt** and call your LLM exactly as you do today.
4. **Era reads the new turn in the background** and updates the user's signals for the next call.

If Era is ever unreachable, `turnContext()` returns an empty string and your agent behaves exactly as it did before Era. The SDK has no dependencies, and the same block format works for support, sales, and companion agents — there is no per-vertical setup.
