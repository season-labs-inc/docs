## Prerequisites

- Node.js 18 or later
- An Era API key and partner ID (contact [ayushman@season.team](mailto:ayushman@season.team))

## Get started

<Steps>
  <Step title="Install the SDK">
    ```bash
    npm install @season/era-sdk
    ```
  </Step>

  <Step title="Set your environment variables">
    ```bash
    ERA_API_KEY=your_api_key
    ERA_URL=https://your-era-endpoint.com
    ERA_PARTNER_ID=your_partner_id
    ```

    Or export them in your shell / `.env.local` file.
  </Step>

  <Step title="Initialize the client">
    ```typescript
    import { Era } from "@season/era-sdk";

    const era = new Era({
      apiKey: process.env.ERA_API_KEY!,
      baseUrl: process.env.ERA_URL!,
      partnerId: process.env.ERA_PARTNER_ID!,
    });
    ```

    Pass `userId` and `sessionId` here if they are known at startup, or per-call inside `turnContext`.
  </Step>

  <Step title="Inject context before each LLM call">
    ```typescript
    // messages is the conversation history so far
    const block = await era.turnContext(messages, {
      userId: "user_abc",
      sessionId: "session_xyz",
    });

    // Append to your system prompt
    const systemPrompt = `You are a helpful assistant.\n\n${block}`;
    ```

    `block` is a plain string — append it however your framework expects. On the very first turn, or if the Era service is unreachable, it returns `""` safely.
  </Step>
</Steps>

## What you get

After the second turn, `block` looks like:

```
[STATE v1] urgency=time_sensitive knowledge=comparing_plans [/STATE]
[GOAL v1] active=seek:find_the_right_plan resolved=partial [/GOAL]
[VALUE v1] top=[cost:high, simplicity:moderate] [/VALUE]
[INTENT v1] intent=comparing(0.9) specificity=specific journey=[curious,comparing] [/INTENT]
[CAUSAL v1] trigger=trial_ending_this_week emotion=hopeful need_level=esteem [/CAUSAL]
[REC v1] action=validate do="Confirm which two plans they are deciding between, then compare directly" priority=0.7 [/REC]
```

Your model sees this on every turn and answers what the user is actually asking — [here's a live example of the difference it makes](/use-cases/fan-companion).
