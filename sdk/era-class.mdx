## Constructor

```typescript
import { Era } from "@season/era-sdk";

const era = new Era(options: EraOptions);
```

See [`EraOptions`](/sdk/types#eraoptions) for the full options reference.

The constructor throws `Error` if `mode` is present but is not `"async"` or `"sync"`.

### Minimal example

```typescript
const era = new Era({
  apiKey: process.env.ERA_API_KEY!,
  baseUrl: process.env.ERA_URL!,
  partnerId: "your_partner_id",
});
```

### Example with defaults

```typescript
const era = new Era({
  apiKey: process.env.ERA_API_KEY!,
  baseUrl: process.env.ERA_URL!,
  partnerId: "your_partner_id",
  visitorId: "era_vid_00000000-0000-4000-8000-000000000000",
  sessionId: "session_xyz",
  mode: "async",
  brandVoice: "concise and direct",
  timeoutMs: 2000,
  maxRetries: 1,
  debug: true,
});
```

## `turnContext(messages, options?)`

Fetches a cognitive context block for the current turn.

```typescript
async turnContext(
  messages: EraMessage[],
  options?: TurnContextOptions,
): Promise<string>
```

### Parameters

<ParamField path="messages" type="EraMessage[]" required>
  Full conversation history through the current user message. See [`EraMessage`](/sdk/types#eramessage).
</ParamField>

<ParamField path="options.visitorId" type="string">
  Browser continuity identifier from the Era Web SDK.
</ParamField>

<ParamField path="options.partnerEndUserId" type="string">
  Your stable identifier for an authenticated user. Supply it only from a trusted backend.
</ParamField>

<ParamField path="options.email" type="string">
  Trusted email bridge assertion. Requires `partnerEndUserId`.
</ParamField>

<ParamField path="options.phone" type="string">
  Trusted phone bridge assertion. Requires `partnerEndUserId`.
</ParamField>

<ParamField path="options.sessionId" type="string">
  Stable identifier for this conversation. It overrides the constructor default. If neither is present, the SDK derives a deterministic ID from the first user message.
</ParamField>

Pass at least `visitorId` or `partnerEndUserId`. If neither is available, the SDK returns `""` without making an Era request.

### Returns

`Promise<string>` containing the cognitive context block to inject into your system prompt.

The method returns `""` when:

- The last normalized message is not from the user.
- Neither `visitorId` nor `partnerEndUserId` is available.
- An expected network, timeout, or Era API failure occurs after retries.

Expected service failures fail open. Programming errors outside those paths can still reject. See [Fail-open design](/concepts/fail-open).

### Usage

```typescript
const block = await era.turnContext(messages, {
  visitorId,
  partnerEndUserId: authenticatedUser.id,
  sessionId,
});

if (block) {
  systemPrompt += "\n\n" + block;
}
```

### Memoization

The latest result is keyed by all identity assertions, `sessionId`, and the current one-based `turn_index`. Repeated calls with the same latest turn can share one request without reusing a block under a different identity. See [Identity and session management](/concepts/sessions#identity-aware-memoization).

## Request format

Under the hood, `turnContext()` sends `POST {baseUrl}/v1/era/turn`:

```json
{
  "partner_id": "your_partner_id",
  "visitor_id": "era_vid_00000000-0000-4000-8000-000000000000",
  "partner_end_user_id": "customer-user-123",
  "email": "person@example.com",
  "session_id": "session_xyz",
  "mode": "async",
  "brand_voice": "concise and direct",
  "turns": [
    { "role": "user", "turn_index": 1, "text": "Hello" },
    { "role": "assistant", "turn_index": 2, "text": "Hi! How can I help?" },
    { "role": "user", "turn_index": 3, "text": "What are your prices?" }
  ]
}
```

Authentication uses the secret `X-API-Key` request header. Keep this key on your trusted backend.
